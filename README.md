## ğŸ§© Note on Table Creation and Data Import

When building the database, itâ€™s best to **create tables and import data step by step** rather than executing the entire SQL script at once.

#### ğŸ’¡ Why Follow This Approach?

If you run all table creation scripts together, tools like **MySQL Workbench** automatically list tables **alphabetically** in the schema view.
While convenient, this can make it difficult to understand the actual data flow and dependencies because:

* The **logical sequence** of relationships (e.g., *Supplier â†’ Products â†’ Orders*) is lost.
* **Foreign key references** become harder to trace.
* **Beginners may struggle** to understand or debug table dependencies and joins.

#### âœ… Recommended Logical Order

To maintain a clear and structured workflow, create and load data in the following order:

1. **Supplier** â†’ Base table for product sourcing
2. **Categories** â†’ Defines product grouping
3. **Employees** â†’ Stores employee details
4. **Customers** â†’ Contains customer data
5. **Products** â†’ Depends on supplier and category
6. **Orders** â†’ Depends on customer and employee
7. **Order_Details** â†’ Depends on orders and products

Following this order ensures that **foreign key relationships are valid**, **data integrity is maintained**, and **troubleshooting becomes easier** throughout development and testing.

##  Business Analysis Questions Covered

This project answers real-world business questions such as:

### 1. Customer Insights
- How many unique customers have placed orders?
- Which customers have placed the highest number of orders?
- What is the total and average purchase value per customer?
- Who are the top 5 customers by total purchase amount?

### 2. Product Performance
- How many products exist in each category?
- What is the average price of products by category?
- Which products have the highest total sales volume (by quantity)?
- What is the total revenue generated by each product?
- How do product sales vary by category and supplier?

### 3. Sales and Order Trends
- How many orders have been placed in total?
- What is the average value per order?
- On which dates were the most orders placed?
- What are the monthly trends in order volume and revenue?
- How do order patterns vary across weekdays and weekends?

### 4. Supplier Contribution
- How many suppliers are there in the database?
- Which supplier provides the most products?
- What is the average price of products from each supplier?
- Which suppliers contribute the most to total product sales (by revenue)?

### 5. Employee Performance
- How many employees have processed orders?
- Which employees have handled the most orders?
- What is the total sales value processed by each employee?
- What is the average order value handled per employee?

### 6. Order Details Deep Dive
- What is the relationship between quantity ordered and total price?
- What is the average quantity ordered per product?
- How does the unit price vary across products and orders?
